Les outils de build

une image (à faire)

Maven

  • les apports positifs :-)
    • standardisation (sources, repos, goals)
    • très répandu
  • le côté sombre :-(
    • leeeeeeeent
    • XML
    • boîte noire
    • difficile à étendre
    • build non incrémental
    • en fait un peu trop à mon goût

Buildr

  • reprend les bons points issus de maven
  • DSL de build
  • ne fait que du build... mais peut faire plus assez facilement
  • JVM : java, scala, groovy, clojure...
  • rake
  • ruby

une image des couches buildr/rake/ruby

Comparatif objectif : les commandes

maven buildr
mvn eclipse:eclipse buildr eclipse
mvn compile -Dmaven.test.skip=true buildr compile test=no
mvn test -Dtest.match="*MonTest.java" buildr test=MonTest

Comparatif objectif : quelques chiffres

maven 1 buildr
Projet A : SLOC
Projet A : temps de build total 12 min 2 min 30
Projet B : SLOC 19183 878
Projet B : nombre fichiers 188 2
Projet B : temps de build total difficile à estimer, mais aux environs de la 1/2h 7 min
Projet B : temps pour faire une release un binome pendant 1/2 journée (avec erreurs) 5 min (safe)

Les défauts de buildr

  • installation difficile (gems)
  • très réactif / moins stable
  • nécessité de traduire les artifacts maven
  • peu adopté

Les voisins

  • gradle
  • sbt
